As the complexity of problem grows, we turn to computer to help us find an acceptable solution. Computer scientists have devised language to model constraint satisfaction problems and have developed methods for solvingthem. The language of constraints is iused to model simple cognitive tasks such as vision, language comprehension, default reasoning and abduction, as well as tasks that require high levels of human expertise such as scheduling, design, diagnosis and temporal and spatial reasoning.

In general, the tasks posed in the language of constraints are computationally intractable (NP-hard), which means that you cannot expect to design algorithms that scale efficiently with the problem size, in all cases. However, it is possible and desirable to identify special properties of a problem class that can accomodate efficient solutions and to deveop general algorithms that are efficient for as many problems as possible.

A constraint satisfaction problem usually has two compoenents:

  • Variables: Objects or items that can take on a variety of values. The set of possilbe values for a given variable is called its domain. For example, in trying to find an acceptable seating arragement for a dinner party, we may choose to see the chairs as our variables, each with the same domain ,which is the list of all guests.

  • Constraints: Constraints are rules that impose a imitaion on the values that a variable or combination of variables, may be assigned. If the host and hostess must sit at the two ends of hte table, then their choices of seats are constrainted. If two feuding guests must not be placed next to or directly opposite one another, then we must include this constraint in our overall problem statement.

A solution is an assignment of a single value from its domain to each variable such that no constraint is violated. A problem may have one, many, or no solutions. A problem that has one or more solutions is satisfiable or consistent. If there is no possible assignment of values to variables that satisfies all the constraints, then the network is unsatisfiable or inconsistent.

Typical tasks over constraint networks are deteremining whether a solution exists, finding one or all solutions, finding one or all solutions, finding partial instantiation can be extended to a full solution, and finding an optimal solution relative to a given cost function. Such tasks are referred to as constraint safistfaction problems (CSPs).

Constraint networks

Let:

  • Constraint network $R$
  • Variables $X = {x_1, ..., x_n}$
  • Domains $D = {D_1, ..., D_n}$
  • Constraints $C = {C_1, ..., C_t}$

Thus a constraint network can be seen as a triple (X, D, C)

A constraint $C_i$ is a relation $R_i$ defined on a subset of variables $S_i$. A relation denotes the variables' simultaneous legal value assignments. $S_i$ is called the scope of $R_i$.
